🎯 Passeja Concursos — Blog V1
Blog automatizado com IA focado no nicho de concursos públicos brasileiros
Next.js 14 Groq + OpenRouter + Gemini Pollinations.ai FLUX Vercel + GitHub Actions Resend Newsletter Zero banco de dados

O Blog V1 é um sistema completo de blog automatizado com IA, construído em Next.js 14 com App Router. Ele gera artigos automaticamente todo dia, possui painel admin completo, simulado com tutor IA, newsletter automática e imagens de capa geradas por IA — tudo funcionando em serverless sem banco de dados.

🌐 Produção: passejaconcursos.com.br

Stack & Tecnologias

⚛️

Next.js 14 (App Router)

Framework principal. SSG para artigos, Server Components, rotas API serverless. Zero banco de dados — CMS é Markdown puro.

🤖

Geração de Conteúdo

Groq (principal, gratuito) → OpenRouter → Gemini em fallback automático. Mesma cadeia usada em artigos, simulado e feedback do tutor.

🎨

Pollinations.ai FLUX

Geração de imagens de capa gratuita, sem chave de API. Resolução 1200×630px. Suporte a image-to-image com modelo kontext.

☁️

Vercel + GitHub

Deploy automático a cada commit. O admin não escreve em disco — commita via GitHub API e a Vercel rebuilda.

📧

Resend

Envio de e-mail para resultados do simulado e newsletter diária automática. Plano gratuito: 100 e-mails/dia.

⚙️

GitHub Actions

Geração diária de artigos às 08:00 BRT e envio de newsletter às 11:00 BRT. Totalmente autônomo após configuração.

Instalação Local

1

Clone e instale dependências

git clone https://github.com/alter012026-hash/Blog_V1
cd Blog_V1
npm install
2

Configure as variáveis de ambiente

Crie o arquivo .env.local na raiz do projeto (veja seção abaixo).

3

Gere os primeiros artigos

npm run generate-article
# ou com tema específico:
node scripts/generate-article.js --topic "concursos INSS 2026" --count 1
4

Rode o servidor de desenvolvimento

npm run dev   # http://localhost:3000

Deploy na Vercel

1

Importe o repositório

Acesse vercel.com, clique em "Add New Project" e selecione o repositório Blog_V1.

2

Configure as variáveis de ambiente

Em Settings → Environment Variables, adicione todas as variáveis listadas acima.

3

Configure o Deploy Hook

Settings → Git → Deploy Hooks → crie um hook chamado "github-actions" → copie a URL para o secret VERCEL_DEPLOY_HOOK no GitHub.

4

DNS no HostGator

No painel HostGator, adicione: registro A apontando para o IP da Vercel e um CNAME www apontando para cname.vercel-dns.com.

⚠️ Filesystem read-only na Vercel: toda operação de escrita no admin (regenerar, remover, editar afiliados) é feita via GitHub API — o código commita diretamente no repositório e a Vercel rebuilda a partir do commit. Nunca tenta escrever em disco.

Conflitos de commit

O GitHub Actions commita artigos automaticamente. Se você fizer push local enquanto o Action está rodando, pode ocorrer conflito. Resolução:

git pull --rebase && git push

A geração de artigos usa a cadeia de provedores Groq → OpenRouter → Gemini com fallback automático. O script monta um prompt otimizado com o tópico, tom e nível de audiência configurados em site.config.js.

Opções do CLI

# Tema aleatório (configurado em site.config.js)
npm run generate-article

# Tema específico
node scripts/generate-article.js --topic "carreira policia federal" --count 1

# Com imagem de referência (ativa modelo kontext — image-to-image)
node scripts/generate-article.js --topic "INSS concurso" --reference-image "https://..."

# Regeneração (reescreve post existente pelo slug)
node scripts/generate-article.js --regenerate "slug-do-post"

Proteções contra duplicatas

O script lê todos os posts existentes e passa os tópicos anteriores ao modelo como contexto, instruindo-o a não repetir ângulos já abordados. Para detectar sobreposições manualmente:

node scripts/detect-topic-overlap.js

Capas com IA (Pollinations.ai FLUX)

Cada artigo gerado recebe automaticamente uma imagem de capa criada pelo modelo FLUX da Pollinations.ai — completamente gratuito, sem necessidade de chave de API.

ConfiguraçãoValor
Resolução1200×630px (proporção ideal OG image)
FormatoJPEG, salvo em public/images/<slug>.jpg
Privacidadeprivate=true — não aparece no feed público
Modelo padrãoFLUX (text-to-image)
Modelo alternativokontext (image-to-image, via --reference-image)
✅ Se a geração de imagem falhar (timeout, API instável), o artigo é salvo normalmente sem capa. A geração de conteúdo nunca é bloqueada por falha de imagem.

Regenerar / Remover posts

Cada post na lista tem botões individuais:

  • Regenerar — reescreve o conteúdo via IA, mantendo slug e URL. Commita o arquivo atualizado no GitHub.
  • Remover — deleta o arquivo .md via GitHub API.

Após o commit, a Vercel rebuilda automaticamente em 1–3 minutos.

Simulado com Tutor IA (/simulado)

Página de simulado com questões geradas on-demand pela IA, no estilo das principais bancas brasileiras: CESPE, FCC, FGV, VUNESP, IBFC.

1

Seleção

Usuário escolhe disciplina (Direito, Raciocínio Lógico, Português etc.) e quantidade de questões.

2

Geração

/api/simulado/questions chama Groq → OpenRouter → Gemini e retorna questões inéditas com gabarito.

3

Resolução

Usuário responde as questões e submete o simulado.

4

Tutor IA

/api/simulado/tutor-feedback gera análise personalizada do desempenho com pontos fortes e fracos.

5

Envio por e-mail

/api/simulado/send-result envia o resultado completo para o e-mail do usuário via Resend.

✅ Fallback automático: se a IA falhar, o simulado usa o banco estático de ~90 questões (lib/quiz-bank.js). A experiência nunca quebra para o usuário.

Indexação Google

O blog possui integração com a Google Indexing API para solicitar indexação de novos artigos automaticamente.

Configuração

1. Crie uma conta de serviço no Google Cloud Console com permissão de Indexing API.
2. Adicione o e-mail da conta de serviço como proprietário no Google Search Console.
3. Configure a variável GOOGLE_SERVICE_ACCOUNT_JSON com o JSON completo da chave.

💡 Bug corrigido: o script de indexação usava JSON.parse() direto na secret do GitHub Actions, o que falhava com JSON multilinha. A correção atual usa extração por regex campo a campo, evitando problemas de parsing.

Troubleshooting

Admin: "Regenerar post" não funciona

Verifique se as variáveis GITHUB_TOKEN e GITHUB_REPO estão configuradas na Vercel. Sem elas, o botão retorna erro explicativo — nunca tenta escrever em disco.

GitHub Actions falha com "Missing GROQ_API_KEY"

Configure o secret em Settings → Secrets and variables → Actions no repositório do GitHub (não no Vercel).

Conflito de push local vs Action

git pull --rebase && git push

Google Indexing API — erro de JSON parsing

O bug de JSON.parse() com secret multilinha foi corrigido. A extração agora usa regex campo a campo. Se reaparecer, verifique se o JSON foi colado sem quebras de linha extras.

Newsletter só entrega para minha própria conta Resend

Verifique seu domínio em resend.com/domains e atualize RESEND_FROM_EMAIL para um endereço desse domínio.

Simulado sem questões da IA (usando banco estático)

Verifique se ao menos uma das chaves GROQ_API_KEY, OPENROUTER_API_KEY ou GEMINI_API_KEY está configurada na Vercel.
