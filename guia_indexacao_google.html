Parte 1 — IndexNow (Bing, Yandex, DuckDuckGo)
1
Gerar a chave IndexNow
Acesse o site abaixo, clique em Generate API Key e copie a chave gerada (ex: a1b2c3d4e5f6...)
bing.com/indexnow/getstarted
2
Publicar o arquivo de verificação
No seu projeto, renomeie o arquivo public/indexnow-key.txt para public/SUA-CHAVE.txt e coloque a chave como conteúdo:
a1b2c3d4e5f6...
O arquivo deve ficar acessível em: https://seusite.com/a1b2c3d4e5f6.txt
3
Adicionar ao GitHub Secrets
No GitHub: Settings → Secrets and variables → Actions → New repository secret
Nome: INDEXNOW_KEY Valor: a1b2c3d4e5f6...
Parte 2 — Google Indexing API
1
Criar projeto no Google Cloud
Acesse o console, crie um projeto novo e ative a API:
console.cloud.google.com
Caminho: APIs e Serviços → Biblioteca → busque Web Search Indexing API → Ativar
2
Criar Service Account e baixar JSON
Caminho: APIs e Serviços → Credenciais → Criar credenciais → Conta de serviço
Após criar, clique na conta → aba Chaves → Adicionar chave → JSON → salve o arquivo no seu computador.
3
Adicionar ao Google Search Console
Acesse o Search Console → Configurações → Usuários e permissões → Adicionar usuário
search.google.com/search-console
Cole o e-mail da Service Account (ex: indexing-bot@projeto.iam.gserviceaccount.com) e defina permissão como Proprietário.
4
Converter JSON para base64
No terminal do Windows (PowerShell), rode:
[Convert]::ToBase64String([IO.File]::ReadAllBytes("C:\caminho\para\arquivo.json"))
Copie o resultado (string longa começando com eyJ...)
5
Adicionar ao GitHub Secrets
Settings → Secrets and variables → Actions → New repository secret
Nome: GOOGLE_SERVICE_ACCOUNT_B64 Valor: eyJ0eXBlIjoic2VydmljZV9hY2NvdW50... (o base64 do passo anterior)
Resumo — secrets necessários no GitHub
INDEXNOW_KEY
Chave gerada em bing.com/indexnow
GOOGLE_SERVICE_ACCOUNT_B64
JSON da Service Account em base64
VERCEL_DEPLOY_HOOK
Já existia — webhook do deploy na Vercel
Usar manualmente (opcional)
Rode no terminal dentro da pasta do projeto:
node scripts/ping-indexing.js --new
Sem flags → pinga todos os posts
--new → só posts das últimas 24h
--url /blog/meu-artigo → URL específica
Automático via GitHub Actions
A cada novo artigo publicado, o workflow aguarda 60s o deploy da Vercel e pinga automaticamente Google + IndexNow. Nenhuma ação manual necessária.
