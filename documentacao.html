📋 Documentação — Rede de Blogs com IA

Tudo que você precisa para operar, escalar e monetizar sua rede de blogs automatizados.

🗺️ Como funciona

🤖

IA gera o conteúdo

Groq, OpenRouter ou Gemini escrevem artigos completos com SEO, frontmatter e markdown limpo automaticamente.

⚡

GitHub Actions publica

Workflow diário faz commit dos artigos novos e dispara o redeploy na Vercel. Você não precisa fazer nada.

💰

Blog gera receita

AdSense exibe anúncios nos artigos. Links de afiliado inseridos automaticamente pelo contexto das palavras-chave.

🌐

Escala para N nichos

Mesmo código, só muda o site.config.js. Cada nicho = novo repo + novo projeto Vercel.

💡
Blog atual: "Aprovado Já" — nicho de concursos públicos. Stack: Next.js 14 + Markdown como CMS + GitHub Actions + Vercel.

🔑 Variáveis de ambiente

Ficam no arquivo .env.local na raiz do projeto. Nunca vai pro Git (está no .gitignore).

# Groq (prioridade 1) — https://console.groq.com
GROQ_API_KEY=gsk_...

# OpenRouter (prioridade 2) — https://openrouter.ai/keys
OPENROUTER_API_KEY=sk-or-v1-...

# Gemini (fallback final) — https://aistudio.google.com/app/apikey
GEMINI_API_KEY=AIza...

# URL do site (para o Next.js)
NEXT_PUBLIC_SITE_URL=https://aprovadoja.com.br

🤖 Script de geração

Arquivo: scripts/generate-article.js

Uso básico

npm run generate-article

Gera 1 artigo com tópico aleatório baseado nas categorias do config.

Tópico específico

node scripts/generate-article.js
--topic "como passar na OAB"
--count 2
✅
Fix aplicado: O script agora sempre salva o artigo com frontmatter completo (title, date, category, excerpt) e limpa markdown inválido gerado pela IA (===, --- soltos).

📄 Formato do frontmatter

Todo arquivo .md em posts/ deve ter este cabeçalho:

---
title: "Título do artigo"
date: "2026-06-14" ← formato YYYY-MM-DD obrigatório
category: "Técnicas de Estudo" ← deve estar nas categories do config
excerpt: "Resumo de até 160 caracteres..."
targetKeyword: "palavra-chave principal"
secondaryKeywords: ["kw2", "kw3"]
readingTime: "" ← calculado automaticamente pelo lib/posts.js
---
⚠️
Nome do arquivo: sempre no formato YYYY-MM-DD-slug-do-titulo.md. Arquivos sem data no nome resultam em "Invalid Date" nos cards.

🚀 Vercel — Deploy

1. Conectar repositório

vercel.com → Add New Project → importe o repositório GitHub → Deploy.

2. Deploy Hook

Settings → Git → Deploy Hooks → crie "github-actions" → copie a URL → adicione como Secret VERCEL_DEPLOY_HOOK no GitHub.

3. Domínio

Settings → Domains → adicione seu domínio. Atualize o DNS no registrador apontando para Vercel.

💰 Google AdSense

ℹ️
O AdSense exige aprovação do site antes de exibir anúncios. Geralmente leva 2-4 semanas e requer conteúdo suficiente (mínimo ~20-30 artigos) e domínio próprio configurado.
EtapaComo fazer
1. Criar contaadsense.google.com com a conta Google
2. Adicionar siteCole a URL do domínio no AdSense
3. Aguardar aprovaçãoContinue gerando conteúdo enquanto aguarda
4. Inserir Publisher IDsite.config.js → adsense.publisherId = ca-pub-XXXXXXXX
5. Inserir Slot IDsadsense.slots.header, inArticle, sidebar
6. DeployCommit + push → anúncios aparecem automaticamente

🔗 Links de afiliado

Configure em site.config.js → affiliates[]. Cada afiliado tem palavras-chave que disparam o link no contexto do artigo.

affiliates: [
  {
    id: "estrategia",
    name: "Estratégia Concursos",
    url: "https://estrategiaconcursos.com.br/?ref=SEU_ID", ← seu link de afiliado
    keywords: ["apostila", "curso", "questões"],
    cta: "Conheça os cursos do Estratégia Concursos",
  },
],
⚠️
CONAR: Declare links de afiliado com aviso "Este post contém links de afiliado" conforme exigido. Inclua no Footer ou no topo dos artigos.
