O negócio em uma frase
Você cria um blog de nicho, usa IA para produzir artigos no piloto automático, o Google envia visitantes orgânicos, e você ganha com anúncios e comissões de afiliado — sem estoque, sem produto, sem atendimento.
As 4 partes do sistema
1
Blog de nicho (Next.js na Vercel)
O site é a sua "loja". Focado num assunto específico — finanças, saúde, pets. Quanto mais nichado, mais fácil ranquear no Google.
2
Geração automática de artigos via IA
Um script chama a API do Claude 3× por dia, gera artigos com SEO otimizado e os salva automaticamente no repositório.
3
Tráfego orgânico pelo Google (SEO)
Artigos bem escritos aparecem nas buscas. Sem pagar por anúncios. Demora 3–6 meses para ganhar tração real.
4
Receita: afiliados + AdSense
Visitante lê o artigo → clica num link de afiliado → você ganha comissão. Ou apenas lê e você ganha pelos anúncios exibidos.
Esse modelo é chamado de "niche site". O diferencial hoje é usar IA para escalar a produção de conteúdo sem custo fixo de equipe.

O que cada arquivo faz
✓
site.config.js — painel de controle
O único arquivo que você edita. Define nicho, nome, afiliados, AdSense. Todo o sistema usa esse arquivo.
✓
scripts/generate-article.js — o gerador
Chama a API do Claude com prompts otimizados para E-E-A-T (padrão anti-penalização do Google), salva o artigo em Markdown na pasta posts/.
✓
.github/workflows/generate-articles.yml — automação
Agenda a geração 3× por dia (9h, 13h, 18h BRT), faz commit e dispara deploy na Vercel.
✓
app/sitemap.js — visibilidade
Gera automaticamente o sitemap.xml com todos os artigos. O Google usa isso para indexar suas páginas.
Passo a passo para publicar
1
Extraia o ZIP e configure
Abra site.config.js e edite com o seu nicho, nome e URL.
name: "Seu Blog", niche: "seu nicho aqui", url: "https://seu-dominio.com.br"
2
Crie conta gratuita na Vercel
Acesse vercel.com. É onde o blog vai ficar hospedado com SSL incluso e deploy automático.
3
Suba o projeto para o GitHub
Crie um repositório e faça o push:
git init git add . git commit -m "primeiro commit" git remote add origin https://github.com/usuario/blog git push -u origin main
4
Conecte o GitHub na Vercel
No painel da Vercel → "Add New Project" → selecione o repositório → Deploy. O Next.js é detectado automaticamente.
5
Adicione a chave da API
Vercel → seu projeto → Settings → Environment Variables:
ANTHROPIC_API_KEY = sk-ant-sua-chave-aqui
Obtenha em console.anthropic.com → API Keys.
6
Gere os primeiros artigos localmente
Teste antes de ativar a automação:
npm install ANTHROPIC_API_KEY=sua_chave npm run generate-article
7
Configure os Secrets no GitHub
GitHub → repositório → Settings → Secrets → Actions → New secret:
ANTHROPIC_API_KEY → sua chave Anthropic VERCEL_DEPLOY_HOOK → URL do Vercel
Deploy Hook: Vercel → Projeto → Settings → Git → Deploy Hooks.
Após o deploy inicial, cada artigo gerado pela pipeline aparece no blog em menos de 2 minutos.

📢 Google AdSense
Pago por visualização e clique. Precisa de volume. Ative com moderação para não poluir.

Hostinger, Shopify, Semrush, Canva Pro — comissões recorrentes mensais são as mais valiosas a longo prazo.

O AdSense exige aprovação manual. O Google analisa o site e pode levar dias ou semanas. Requisitos: conteúdo original, site funcional, sem violações de políticas.
O que esperar em cada fase
Semana 1 — Setup
Configure o blog, suba na Vercel, ative a pipeline. Publique os primeiros 20 artigos manualmente para dar volume inicial ao Google.
Mês 1–2 — Invisibilidade
O Google ainda não conhece o site. Zero tráfego orgânico. A pipeline continua gerando. Envie o sitemap para o Search Console.
Mês 3–4 — Primeiros sinais
Primeiras impressões no Search Console. Artigos aparecem em posições 20–50. Sem receita ainda, mas o sinal está chegando.
Mês 5–6 — Tráfego real
Artigos sobem para a primeira página. Primeiros visitantes orgânicos. Primeiras comissões de afiliado aparecem.
Mês 7–12 — Crescimento composto
Com 300–500+ artigos, o tráfego cresce exponencialmente. Otimize os artigos na página 2 do Google para subir mais rápido.
Mês 12+ — Replicação
Replique o modelo em outro nicho. Mesmo código, diferente site.config.js. Cada site tem potencial de receita independente.
Fatores de sucesso
✅ Aceleram o crescimento
• Nicho com baixa concorrência
• Artigos longos (1.500+ palavras)
• Palavras-chave de cauda longa
• Search Console ativo desde o dia 1
• Revisão periódica dos artigos gerados
⚠️ Atrasam ou penalizam
• Nicho saturado (finanças genérico)
• Artigos curtos e rasos
• Conteúdo repetitivo ou duplicado
• Site sem identidade clara
• Não enviar o sitemap ao Google
Por que o Search Console é essencial
O Google Search Console é o painel gratuito do Google que mostra quais artigos seus estão aparecendo nas buscas, em que posição, quantos cliques você recebe — e quais páginas estão sendo ignoradas. Sem ele você publica no escuro.

Passo a passo para ativar
🌐
1. Acesse o Search Console
Vá em search.google.com/search-console e entre com sua conta Google. Clique em "Adicionar propriedade".
🏷️
2. Escolha o tipo de propriedade
Selecione "Prefixo de URL" (mais fácil). Digite a URL exata do seu blog: https://seu-dominio.com.br
✅
3. Verifique a propriedade via HTML
O Google vai te dar uma metatag. Copie o código de verificação (parece com abc123xyz) e cole em site.config.js:
// site.config.js → seção seo: googleSiteVerification: "abc123xyz-seu-codigo-aqui"
Faça deploy → volte ao Search Console → clique em Verificar.
🗺️
4. Envie o Sitemap
No menu lateral do Search Console → Sitemaps → campo "Adicionar novo sitemap" → digite:
sitemap.xml
Clique em Enviar. O Google vai agora descobrir todos os seus artigos automaticamente.
🔄
5. Solicite indexação dos primeiros artigos (opcional)
Menu lateral → Inspeção de URL → cole a URL de um artigo → clique em "Solicitar indexação". Faça isso nos primeiros 10–20 artigos para acelerar.
📈
6. Aguarde 3–7 dias e analise
Vá em Desempenho → Resultados da Pesquisa. Você vai ver impressões, cliques, CTR e posição média por página e por palavra-chave.
O que monitorar semanalmente
📍
Artigos na posição 8–20 (ouro escondido)
Esses artigos estão quase na primeira página. Uma pequena melhoria no conteúdo pode dobrar o tráfego. Priorize eles.
🔍
Palavras-chave reais que trouxeram visitas
O Google mostra exatamente o que as pessoas digitaram para chegar no seu site. Use essas palavras para criar novos artigos relacionados.
⚠️
Erros de cobertura
Menu Cobertura mostra páginas que o Google tentou indexar mas encontrou erro. Corrija imediatamente — cada erro é dinheiro perdido.
Após 60–90 dias com o Search Console ativo, você vai ter dados suficientes para saber quais artigos têm potencial e onde focar a energia de otimização.
