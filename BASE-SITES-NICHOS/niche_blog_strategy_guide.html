O negócio em uma frase

Você cria um blog de nicho, usa a IA para produzir artigos de qualidade no piloto automático, o Google envia visitantes orgânicos, e você ganha dinheiro com anúncios e comissões de afiliado — sem estoque, sem produto, sem atendimento.

As 4 partes do sistema

1
Blog de nicho (Next.js)
O site é a sua "loja". Focado em um assunto específico — finanças, saúde, pets, produtividade. Quanto mais nichado, mais fácil de ranquear no Google.
2
Geração de artigos via IA
Um script chama a API do Claude 3x por dia, gera artigos com SEO otimizado e os salva automaticamente no repositório.
3
Tráfego orgânico (SEO)
Artigos bem escritos sobre termos que as pessoas buscam aparecem no Google. Sem pagar por anúncios. Demora 3–6 meses para começar a aparecer.
4
Receita (afiliados + AdSense)
Visitante lê o artigo → clica num link de afiliado → você ganha comissão. Ou apenas lê a página e você ganha pelos anúncios exibidos.
Esse modelo é chamado de "niche site". Existem pessoas ganhando de R$500 a R$50.000/mês com ele. O diferencial hoje é usar IA para escalar a produção de conteúdo.

Arquivos do projeto e o que cada um faz

✓
site.config.js — o painel de controle
O único arquivo que você edita. Define o nicho, nome do blog, links de afiliado, configurações de AdSense. Todo o resto usa esse arquivo.
✓
scripts/generate-article.js — o gerador
Chama a API do Claude com um prompt otimizado para E-E-A-T (padrão do Google), recebe o artigo em Markdown e salva na pasta posts/.
✓
.github/workflows/generate-articles.yml — a automação
Agenda a geração 3x por dia (9h, 13h, 18h BRT). Faz commit dos artigos e dispara o deploy na Vercel automaticamente.
✓
app/sitemap.js — visibilidade no Google
Gera automaticamente o sitemap.xml com todos os artigos publicados. O Google usa isso para descobrir e indexar suas páginas.

Passo a passo para publicar

1
Extraia o ZIP e configure
Baixe o ZIP gerado, extraia e abra a pasta no seu editor. Edite site.config.js com o seu nicho.
// site.config.js name: "Seu Blog", niche: "seu nicho aqui", url: "https://seu-dominio.com.br",
2
Crie uma conta na Vercel (grátis)
Acesse vercel.com e crie uma conta gratuita. É onde seu blog vai ficar hospedado.
3
Suba para o GitHub
Crie um repositório no GitHub e faça o push do projeto. Pode ser repositório público ou privado.
git init git add . git commit -m "primeiro commit" git remote add origin https://github.com/seu-usuario/seu-blog git push -u origin main
4
Conecte o GitHub na Vercel
No painel da Vercel → "Add New Project" → selecione o repositório → clique em Deploy. A Vercel detecta o Next.js automaticamente.
5
Adicione a chave da API nas variáveis de ambiente
No painel da Vercel → seu projeto → Settings → Environment Variables. Adicione:
ANTHROPIC_API_KEY = sua_chave_aqui
Obtenha a chave em console.anthropic.com → API Keys.
6
Gere os primeiros artigos localmente
Antes de ativar a automação, teste gerando artigos na sua máquina:
npm install ANTHROPIC_API_KEY=sua_chave npm run generate-article
7
Configure os Secrets no GitHub
No GitHub → seu repositório → Settings → Secrets and variables → Actions → New secret:
ANTHROPIC_API_KEY → sua chave da Anthropic VERCEL_DEPLOY_HOOK → URL gerada na Vercel
O Deploy Hook fica em: Vercel → Projeto → Settings → Git → Deploy Hooks.
Após o deploy inicial, todo novo artigo gerado pelo GitHub Actions vai aparecer automaticamente no seu blog em 1–2 minutos.

Como a pipeline funciona

Horários de execução (automático)
09:00 BRT
13:00 BRT
18:00 BRT
A cada execução, 1 artigo é gerado. 3 artigos por dia. ~90 artigos por mês. ~1.000 artigos por ano.
O que acontece em cada execução
1
GitHub Actions acorda o workflow
2
Lê os artigos já publicados para não repetir temas
3
Pede ao Claude para gerar um tópico com alta busca no nicho
4
Pede um artigo completo (1.200+ palavras) com SEO e links de afiliado contextuais
5
Salva o arquivo .md, faz commit e push no GitHub
6
Dispara deploy na Vercel → artigo no ar em ~1 minuto

Disparar manualmente (quando quiser)

GitHub → repositório → aba Actions → "Gerar e Publicar Artigos" → Run workflow
Opções no disparo manual: - Tópico específico: "como guardar dinheiro no Brasil" - Quantidade: 1, 2 ou 3 artigos por execução
Custo estimado de API: cada artigo custa ~$0.01–0.02 USD com Claude Sonnet. 90 artigos/mês = ~$1.50–2.00/mês.

Fontes de receita

Afiliados
Comissão por conversão. Melhor fonte de renda. Funciona mesmo sem muito tráfego se o nicho for bem escolhido.
Renda variável
Google AdSense
Pago por visualização/clique em anúncios. Precisa de volume de tráfego. Ative com moderação.
Renda passiva

Plataformas de afiliado por nicho

Finanças pessoais
Nubank, XP Investimentos, Rico, Banco Inter, Easynvest — comissão por conta aberta ou valor investido. CPC alto.
Saúde e bem-estar
iHerb, Amazon, Hotmart (cursos de nutrição, yoga). Ticket médio alto em suplementos.
Tecnologia e SaaS
Hostinger, Shopify, Semrush, Canva Pro — comissões recorrentes mensais são as melhores.
Cursos e infoprodutos
Hotmart, Kiwify, Eduzz, Monetizze — 20–50% de comissão por venda. Se o nicho for educação, é o melhor caminho.
Os links de afiliado já estão configurados no site.config.js. O script de geração instrui o Claude a inseri-los de forma natural e contextual — sem forçar a barra.
O AdSense exige aprovação manual. O Google analisa o site e pode levar dias/semanas. Requisitos: conteúdo original, site funcional, sem violações de políticas.

O que esperar em cada fase

Semana 1 — Setup
Configure o blog, suba na Vercel, ative a pipeline. Publique os primeiros 10–20 artigos manualmente para dar volume inicial.
Mês 1–2 — Invisibilidade
O Google ainda não conhece o site. Zero tráfego orgânico. A pipeline continua gerando artigos. Envie o sitemap para o Google Search Console.
Mês 3–4 — Primeiros sinais
Primeiras impressões no Search Console. Artigos começam a aparecer em posições 20–50. Sem receita ainda, mas o sinal está chegando.
Mês 5–6 — Tráfego real
Artigos sobem para as primeiras páginas. Primeiros visitantes orgânicos. Primeiras comissões de afiliado aparecem.
Mês 7–12 — Crescimento
Com 300–500+ artigos publicados, o tráfego cresce de forma composta. Otimize os artigos que já estão na página 2 do Google.
Mês 12+ — Replicação
Replique o modelo em outro nicho. Mesmo código, diferente site.config.js. Cada site adicional tem custo mínimo e potencial de receita independente.

O que acelera o processo

Fazem crescer mais rápido
• Nicho com baixa concorrência
• Artigos longos (+1.500 palavras)
• Palavras-chave de cauda longa
• Google Search Console ativo
• Revisar artigos gerados periodicamente
O que pode atrasar
• Nicho saturado (finanças genérico)
• Artigos curtos e rasos
• Conteúdo duplicado ou genérico
• Site sem identidade clara
• Não enviar o sitemap ao Google
