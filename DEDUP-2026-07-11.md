# Limpeza de duplicatas — 2026-07-11

## Causa raiz (o motivo real dos duplicados)

O pipeline **já tinha** um bom sistema anti-duplicata (`lib/quality-engine.js`,
similaridade de Jaccard, pool de seeds grande, log de qualidade). O problema
não era esse sistema — era que ele estava **cego** desde a criação:

No `.github/workflows/generate-articles.yml`, o passo "Check changes" fazia:

```bash
git add posts/ public/images/
```

Ou seja: `posts/` e `public/images/` eram commitados, mas
`.content-signatures.json`, `.used-topics.json` e `.quality-log.json`
**nunca eram salvos no Git**. Como cada execução do GitHub Actions começa
com um checkout limpo do repositório, esses arquivos voltavam a ficar vazios
(ou desatualizados) a cada dia. Resultado: o comparador de similaridade
rodava sempre contra uma lista praticamente em branco, e o pipeline não
"lembrava" dos últimos posts publicados de um dia para o outro.

**Corrigido**: o workflow agora também dá `git add` nesses três arquivos, então
a memória de deduplicação passa a persistir entre execuções.

## O que mais foi corrigido

- **21 posts** (de 74) nunca tiveram assinatura de conteúdo salva — foram
  gerados antes desse controle existir, ou foram gerados no mesmo dia que
  seus duplicados e a assinatura não foi persistida. Fiz o backfill das
  assinaturas de todos os posts restantes, para que o comparador tenha o
  histórico completo daqui pra frente.
- **1 colisão de slug real**: dois arquivos com nomes de datas diferentes
  geravam a mesma URL (`como-organizar-rotina-de-estudos-para-concurso-dicas-praticas`),
  o que podia causar comportamento imprevisível na geração de páginas
  estáticas. Mantido o mais completo, removido o outro.
- **7 arquivos "fantasmas"**: já existiam redirects 301 configurados em
  `next.config.js` apontando para outro post (de uma limpeza anterior), mas
  o arquivo `.md` de origem nunca tinha sido apagado de `posts/`. Isso
  significa que o conteúdo duplicado continuava sendo lido pelo
  `getAllPosts()`, aparecendo no sitemap, no índice de busca interno e nas
  páginas de categoria — mesmo com o redirect ativo. Removidos.
- **5 clusters novos de duplicata** identificados por similaridade de
  Jaccard + mesmo tema, sem redirect nenhum ainda:
  - `cronograma-de-estudos-para-concurso-em-6-meses` → mesclado com o guia
    principal de cronograma
  - `tecnicas-para-questoes-de-matematica-financeira-em-concursos` (mesmo
    dia, mesmo tema de `tecnicas-avancadas-...`) → mesclado
  - `melhores-apostilas-gratas-...` (erro de digitação de "gratuitas") →
    mesclado com a versão correta
  - `concursos-publicos-com-mais-vagas-para-nivel-medio` (só 598 palavras,
    bem raso) → mesclado com a versão mais completa
  - `diferencas-entre-cargos-...-o-que-mudou-e` (terceira variação do mesmo
    tema) → mesclado

**Total: 13 posts removidos, 5 redirects 301 novos adicionados** (mais os
17 que já existiam de uma limpeza anterior — 22 no total agora).

## O que fazer depois de subir isso

1. Faça o deploy normalmente (`git push`, Vercel builda sozinho).
2. Depois do deploy, no Google Search Console, use a ferramenta de
   **Inspeção de URL → Solicitar remoção** para as URLs antigas removidas
   (mesmo com redirect 301, acelera a atualização do índice).
3. Envie o sitemap novamente no Search Console para o Google recrawlear.
4. A partir de agora, os próximos `git pull` já vão trazer
   `.content-signatures.json` atualizado — não delete esse arquivo por
   engano, ele é a memória do sistema anti-duplicata.
